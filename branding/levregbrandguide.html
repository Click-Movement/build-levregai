05 — Usage Guidelines

Do's & Don'ts

Consistent application of these rules ensures the LevReg.AI brand remains coherent, accessible, and professional across all touchpoints — from the product UI to marketing materials.

✓
Do
  • Use #14B870 exclusively for the logo, primary CTA buttons, key highlights, and important icons that require immediate attention.
  • Keep most page and app backgrounds dark, using #05070A or #060711 as the base layer.
  • Write primary body text in #F9FAFB (off-white) and secondary/meta text in #9CA3AF (cool gray) for clear visual hierarchy.
  • Reserve #0EA5E9 (cyan blue) for hyperlinks, data visualizations, and secondary CTAs so it complements — not competes with — the green.
  • Use semantic state colors (#22C55E, #F59E0B, #EF4444) strictly for system feedback: success, warning, and error states.
  • Apply #1F2937 for all borders, dividers, and input outlines to maintain subtle structure without visual noise.
  • Use CSS design tokens (--color-primary, etc.) in all code to ensure single-source-of-truth color management.
✕
Don't
  • Do not use #14B870 as a background fill for large areas — it should remain a high-signal accent, not a dominant surface color.
  • Do not place the brand green and cyan blue (#0EA5E9) in direct competition as equal-weight CTAs on the same screen.
  • Do not use light or white backgrounds in the product UI — the palette is dark-first and loses its identity on light surfaces.
  • Do not use warning amber or error red for decorative or non-semantic purposes; these colors carry meaning and misuse erodes trust.
  • Do not use low-contrast text combinations — never place #9CA3AF on #111827 for body copy; use it only for supplementary metadata.
  • Do not introduce new colors outside this palette without formal brand approval — consistency is critical for a regulatory-grade product.
  • Do not use pure white (#FFFFFF) or pure black (#000000) — the off-white and near-black tokens are intentionally chosen for warmth and reduced eye strain.

06 — Accessibility

Contrast Ratios

All primary text and interactive element combinations meet or exceed WCAG 2.1 AA standards (minimum 4.5:1 for normal text, 3:1 for large text and UI components). The dark-first palette is inherently well-suited to high-contrast rendering.
